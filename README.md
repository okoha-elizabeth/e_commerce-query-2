# E-Commerce SQL Analysis

# Overview
This project focuses on analyzing an e-commerce dataset using SQL to extract meaningful business insights. The analysis is centered on understanding revenue trends, customer distribution, and overall business performance.

# Objectives
The goal of this project is to:

1. Transform raw transactional data into structured insights
2. Answer key business questions using SQL
3. Demonstrate the use of joins, filtering, and aggregation in real-world scenarios
   
# Dataset
The dataset consists of:

1. Orders Table: Contains transaction-level data such as order ID, order date, revenue, and location
2. Customers Table: Contains customer-related information such as city and customer ID

# Concepts Covered
## Joins
Combined multiple tables to create a unified dataset

## Advanced Filtering
1. Applied conditional logic to refine datasets
2. Used WHERE, HAVING, and date-based filters to isolate relevant records

## Aggregation & Grouping
1. Used GROUP BY and aggregate functions (SUM, COUNT, AVG)
2.Structured data for meaningful summaries

# Business Questions Answered
## Monthly Revenue
1. Calculated total revenue generated per month
2. Identified revenue trends over time

## Top Performing Month
1. Determined the month with the highest revenue
2. Highlighted peak business performance period

## Top City by Revenue
1. Identified the city generating the highest revenue

# Key Insights
1. Revenue patterns show variation across different months with January as the top preforming month
2. Canada contributes significantly more to overall revenue

# Tools Used
SQL (PostgreSQL / MySQL compatible)

# How to Use
1. Load the dataset into your SQL environment
2. Run the queries provided in the .sql file
3. Modify filters or groupings to explore additional insights

# Conclusion
This analysis demonstrates how SQL can be used to move beyond raw data and provide structured, decision-ready insights that support business growth.
